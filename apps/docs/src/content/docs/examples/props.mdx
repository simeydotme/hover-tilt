## Tilt Factors

Control the intensity of the tilt effect with `tiltFactor` and `tiltFactorY`.
The `tiltFactor` controls tilt on both axes, while `tiltFactorY` overrides control of the vertical tilt.

<TiltDemo client:only="svelte" />

<Tabs syncKey="api">
	<TabItem label="Svelte" icon="seti:svelte">
		```svelte

    	<!-- Default tilt -->
    	<HoverTilt>
    	  <div class="card">tiltFactor: 1</div>
    	</HoverTilt>

    	<!-- Increased tilt -->
    	<HoverTilt tiltFactor={2}>
    	  <div class="card">tiltFactor: 2</div>
    	</HoverTilt>

    	<!-- Different Y tilt -->
    	<HoverTilt tiltFactor={1.5} tiltFactorY={0.5}>
    	  <div class="card">tiltFactor: 1.5, tiltFactorY: 0.5</div>
    	</HoverTilt>
    	```
    </TabItem>
    <TabItem label="Web&nbsp;Component" icon="seti:html">
    	```html
    	<!-- Default tilt -->
    	<hover-tilt>
    	  <div class="card">tilt-factor: 1</div>
    	</hover-tilt>

    	<!-- Increased tilt -->
    	<hover-tilt tilt-factor="2">
    	  <div class="card">tilt-factor: 2</div>
    	</hover-tilt>

    	<!-- Different Y tilt -->
    	<hover-tilt tilt-factor="1.5" tilt-factor-y="0.5">
    	  <div class="card">tilt-factor: 1.5, tilt-factor-y: 0.5</div>
    	</hover-tilt>
    	```
    </TabItem>

</Tabs>

## Spring Animations

Customize the animation physics with `springOptions` and `tiltSpringOptions`.
Control stiffness & damping for different feeling interactions.

- A stiffer spring will create more pronouced elastic effects (more 'boing-y')
- More damping will smooth out the motion at the start and end (less 'zippy')

<SpringDemo client:only="svelte" />

<Tabs syncKey="api">
	<TabItem label="Svelte" icon="seti:svelte">
		```svelte

    	<!-- Default spring -->
    	<HoverTilt scaleFactor={1.1} >
    	  <div class="card"></div>
    	</HoverTilt>

    	<!-- Fast spring -->
    	<HoverTilt scaleFactor={1.1} springOptions={{ stiffness: 0.4, damping: 0.3 }}>
    	  <div class="card"></div>
    	</HoverTilt>

    	<!-- Slow spring -->
    	<HoverTilt scaleFactor={1.1}  springOptions={{ stiffness: 0.05, damping: 0.9 }}>
    	  <div class="card"></div>
    	</HoverTilt>
    	```
    </TabItem>
    <TabItem label="Web&nbsp;Component" icon="seti:html">

    		<Aside type="tip" title="Web Components">
    			**`springOptions`** and **`tiltSpringOptions`** require JSON string values.
    		</Aside>

    	```html
    	<!-- Default spring -->
    	<hover-tilt scale-factor="1.1">
    	  <div class="card"></div>
    	</hover-tilt>

    	<!-- Fast spring (requires JSON string) -->
    	<hover-tilt scale-factor="1.1" spring-options='{"stiffness": 0.4, "damping": 0.3}'>
    	  <div class="card"></div>
    	</hover-tilt>

    	<!-- Slow spring (requires JSON string) -->
    	<hover-tilt scale-factor="1.1" spring-options='{"stiffness": 0.05, "damping": 0.9}'>
    	  <div class="card"></div>
    	</hover-tilt>
    	```
    </TabItem>

</Tabs>

### Tilt Spring options

The `springOptions` will apply to both the scale and tilt animations by default. If you want to apply different physics to the tilt animation, you can use the `tiltSpringOptions` prop.

<TiltSpringDemo client:only="svelte" />

<Tabs syncKey="api">
	<TabItem label="Svelte" icon="seti:svelte">
		```svelte

    	<!-- Default spring -->
    	<HoverTilt scaleFactor={1.1} >
    	  <div class="card"></div>
    	</HoverTilt>

    	<!-- Fast spring -->
    	<HoverTilt scaleFactor={1.1} springOptions={{ stiffness: 0.08, damping: 0.15 }} tiltSpringOptions={{ stiffness: 0.1, damping: 0.2 }}>
    	  <div class="card"></div>
    	</HoverTilt>

    	<!-- Slow spring -->
    	<HoverTilt scaleFactor={1.1}  springOptions={{ stiffness: 0.05, damping: 0.5 }} tiltSpringOptions={{ stiffness: 0.1, damping: 0.075 }}>
    	  <div class="card"></div>
    	</HoverTilt>
    	```
    </TabItem>
    <TabItem label="Web&nbsp;Component" icon="seti:html">

    		<Aside type="tip" title="Web Components">
    			**`springOptions`** and **`tiltSpringOptions`** require JSON string values.
    		</Aside>

    	```html
    	<!-- Default spring -->
    	<hover-tilt scale-factor="1.1">
    	  <div class="card"></div>
    	</hover-tilt>

    	<!-- Bouncey Scale, Soft Tilt -->
    	<hover-tilt scale-factor="1.1" spring-options='{"stiffness": 0.08, "damping": 0.15}' tilt-spring-options='{"stiffness": 0.1, "damping": 0.2}'>
    	  <div class="card"></div>
    	</hover-tilt>

    	<!-- Soft Scale, Elastic Tilt -->
    	<hover-tilt scale-factor="1.1" spring-options='{"stiffness": 0.05, "damping": 0.5}' tilt-spring-options='{"stiffness": 0.1, "damping": 0.075}'>
    	  <div class="card"></div>
    	</hover-tilt>
    	```
    </TabItem>

</Tabs>

## Enter Delay

Control how long the component waits before activating and animating when the cursor enters. Can be used to prevent flickering when the cursor briefly enters the element.

<Tabs syncKey="api">
	<TabItem label="Svelte" icon="seti:svelte">
		```svelte

    	<!-- Default enter delay (0ms) -->
    	<HoverTilt>
    	  <div class="card">Default</div>
    	</HoverTilt>

    	<!-- Longer enter delay -->
    	<HoverTilt enterDelay={300}>
    	  <div class="card">enterDelay: 300ms</div>
    	</HoverTilt>
    	```
    </TabItem>
    <TabItem label="Web&nbsp;Component" icon="seti:html">
    	```html
    	<!-- Default enter delay (0ms) -->
    	<hover-tilt>
    	  <div class="card">Default</div>
    	</hover-tilt>

    	<!-- Longer enter delay -->
    	<hover-tilt enter-delay="300">
    	  <div class="card">enter-delay: 300ms</div>
    	</hover-tilt>
    	```
    </TabItem>

</Tabs>

<EnterDelayDemo client:only="svelte" />

<Aside type="note" title="Enter + Exit Delay">
  **`enterDelay`** respects the `exitDelay` and resting time of the component. So if the user has previously activated
  and then re-enters the element before the component has come to rest; it will immediately re-activate without waiting
  for the `enterDelay` again.
</Aside>
