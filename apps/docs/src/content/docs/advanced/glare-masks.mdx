## When to reach for masks

Glare masks let you constrain the animated highlight to very specific shapes—foil borders, emboss lines, or gradients that only reveal part of the card. Pair masks with `glareMaskMode` and `glareMaskComposite` to emulate multi-layer trading-card finishes.

<Steps>

1. Export or create monochrome PNG/SVG assets representing the regions where the glare should appear.
2. Reference those assets in the `glareMask` prop (comma-delimit multiple masks).
3. Tune `glareMaskMode` (`match-source`, `luminance`, or `alpha`) so the browser interprets the mask correctly.
4. When stacking more than one mask, set `glareMaskComposite` (`add`, `intersect`, etc.) to define how they merge.

</Steps>

<Aside type="caution" title="CORS matters">
  Remote mask URLs must allow cross-origin image usage. Host masks in the same site (e.g., `/img/mask.png`) or ensure
  the CDN sends permissive CORS headers, otherwise the mask will silently fail.
</Aside>

### Reference

- `glareMask` accepts any valid [`mask-image`](https://developer.mozilla.org/en-US/docs/Web/CSS/mask-image) value(s). Comma-separated entries are evaluated left-to-right.
- `glareMaskMode` defaults to `match-source`, which follows each source image’s alpha channel. Use `luminance` for black/white PNGs or `alpha` for pre-multiplied assets.
- `glareMaskComposite` matches CSS [`mask-composite`](https://developer.mozilla.org/en-US/docs/Web/CSS/mask-composite). `intersect` is perfect for combining etched outlines with foil fills, while `add` simply stacks them.
